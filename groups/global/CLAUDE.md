# NanoCures Shared Memory

This global memory applies across NanoCures groups.

## Scientific Standards

- Prefer high-quality reviews, primary literature, regulators, WHO, NIH, FDA, EMA, ClinicalTrials.gov, PubMed, and major journals.
- Label speculative work as speculative.
- Separate target validation, design, ADMET, and translation stages.
- Record dates for time-sensitive facts.

## Security Standards

- Public literature and approved non-PHI data are acceptable mounts.
- PHI, secrets, API keys, raw credential files, and broad home-directory mounts are not acceptable.
- Non-main groups should stay read-only for additional mounts.

## Campaign Defaults

- Optimize for reproducible artifacts over chat-only conclusions.
- Use JSON artifacts for plans, runs, portfolios, and trial ledgers when repeatability matters.
- Keep a short next-actions list in each run artifact.
